# Release 0.3.0

Date: 2026-06-23
Summary: perf + stability + data-quality hardening from architecture review

## Changes

- Write path: per-run FK/content-type cache, single-scan delete index, run
  transaction + per-op savepoints, lean `update_fields` saves, single error key.
- Failure visibility: persist run history on failed runs; do not advance the
  snapshot baseline on failure; record hard API errors.
- Data quality: location-variant dedup (ST/STREET → one site); reconcile
  max-delete-fraction safeguard.
- Resilience: retry backoff + Retry-After; ndjson errors as ForwardClientError;
  tier dependency cycle guard.
- SSoT: render-safe `sync.diff` (no View Diff crash); DB-tolerant target load.
